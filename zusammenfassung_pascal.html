Eine neue Programmiersprache — aber nicht für Menschen

Ein Überblick für Nicht-Programmierer darüber, was wir hier eigentlich machen und warum es wichtig ist.

Das Problem

Du kennst ChatGPT, Claude und Co. — KI-Modelle, die auch Programmcode schreiben können. Das machen sie inzwischen richtig viel: Ein großer Teil aller neuen Software wird schon heute von KI mitgeschrieben oder komplett generiert.

Das Kuriose dabei: Die Programmiersprachen, die diese KIs benutzen (Python, JavaScript, etc.), wurden alle für Menschen entworfen. Für menschliche Augen, menschliche Lesegewohnheiten, menschliche Tippfehler-Toleranz.

Das ist ungefähr so, als würde man einem Roboter-Chirurgen ein Skalpell geben, das für menschliche Hände geformt ist. Es funktioniert — aber es ist nicht optimal.

Die Idee

Was wäre, wenn man eine Programmiersprache baut, die ausschließlich für KI-Agenten gedacht ist?

Nicht zum Lesen für Menschen. Nicht zum Tippen auf einer Tastatur. Sondern optimiert für die Art, wie ein KI-Modell "denkt", Code erzeugt und Fehler vermeidet.

Was wir recherchiert haben

Wir haben uns existierende Sprachen, Typsysteme und Verifikations-Ansätze angeschaut und die besten Ideen destilliert:

🔒

Intelligente Datentypen

Stell dir vor, der Computer weiß nicht nur "das ist eine Zahl", sondern auch "das ist ein Betrag in Euro" oder "das ist eine Temperatur in Celsius". Wenn die KI dann versehentlich Euro mit Kilogramm verrechnen will, sagt der Compiler sofort: Stopp, das ergibt keinen Sinn. So verhindert man eine ganze Kategorie von Fehlern, bevor das Programm überhaupt läuft.

✅

Eingebaute Versprechen (Verträge)

Jede Funktion im Code muss explizit sagen: "Ich erwarte X als Eingabe" und "Ich verspreche Y als Ergebnis." Ein mathematischer Beweiser prüft automatisch, ob diese Versprechen eingehalten werden. Aktuelle Forschung zeigt: KIs können solche Spezifikationen schon mit ~98% Trefferquote selbst schreiben. Das heißt: Die KI schreibt den Code UND den Beweis, dass er korrekt ist.

⚡

Effekt-System ("Was darf diese Funktion?")

Jede Funktion deklariert, ob sie aufs Internet zugreift, Dateien schreibt oder schiefgehen kann. Das steht direkt in der Signatur. So weiß jeder sofort, was ein Code-Baustein tut — ohne reinschauen zu müssen.

🧮

Token-Effizienz ("Jedes Zeichen zählt")

KI-Modelle haben ein begrenztes "Arbeitsfenster" (Kontextfenster). Jedes überflüssige Zeichen im Code verschwendet Platz. Unsere Sprache ist darauf optimiert, mit möglichst wenig Zeichen möglichst viel Bedeutung zu transportieren. Ziel: >50% weniger Platzverbrauch als Python.

🏗️

Code als Struktur, nicht als Text

Für Menschen ist Code eine Textdatei. Für die KI ist es intern ein Syntaxbaum (wie ein Stammbaum, nur für Programmlogik). Unsere Sprache arbeitet direkt mit dieser Baumstruktur, was Syntaxfehler praktisch unmöglich macht.

Was ist daran neu?

Einzelne Teile dieser Ideen gibt es schon — aber keines der existierenden Projekte bringt sie alle zusammen:

Idee Gibt's schon wo? Was fehlt?
Intelligente Typen F# (Microsoft) Nicht für KI, nur für Physik-Einheiten
Automatische Beweise Dafny (Microsoft) Nicht KI-optimiert
Code als Baum Unison Inkompatibel mit allen anderen Tools
Token-optimierte Syntax GlyphLang Nur ein Nischenprojekt
Alles zusammen Niemand ← Das ist unsere Lücke

TL;DR — Kurzfassung

Ich forsche daran, wie eine Programmiersprache aussehen müsste, die nicht für Programmierer, sondern für KI-Agenten gebaut ist. Eine Sprache, in der die KI weniger Fehler macht, sich selbst beweisen kann, dass ihr Code korrekt ist, und weniger "Denkkapazität" für unnötigen Boilerplate verschwendet.

Es ist Grundlagenforschung — wir sind in der Phase, wo wir sammeln, was es gibt, was funktioniert, und was unsere eigene Innovation wäre. Der nächste Schritt wäre, einen ersten Prototyp zu designen.
